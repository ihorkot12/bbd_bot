# Реєстрація членів клубу

Повна форма реєстрації потрібна для чинних і майбутніх членів Black Bear Dojo. Її можуть заповнювати батьки для 1-3 дітей або дорослий учасник для себе.

## Як це працює

1. Батьки заповнюють одну форму на сім'ю.
2. Якщо дітей кілька, у формі заповнюються блоки `Дитина 1`, `Дитина 2`, `Дитина 3`.
3. Бот створює окремого учасника в `members` для кожної дитини.
4. Дорослий учасник заповнює блок дорослого, і бот створює дорослого учасника.
5. Дата народження одразу потрапляє в модуль ДН, а згода на привітання керує тим, чи бот готує привітання.
6. Прямий режим читає Google Form через Forms API. Якщо Forms API недоступний, бот пробує прочитати пов'язані листи відповідей у Google Sheets.
7. Повторні синхронізації не дублюють анкету: бот шукає вже імпортований `response_id` у нотатках учасника.

## Жива Google Form

Поточна форма: `Black Bear Dojo — повна реєстрація учасника та батьків`.

Обов'язковий мінімум полів:

| Поле | Для чого |
| --- | --- |
| `Тип учасника` | Дитина / дорослий учасник |
| `ПІБ дитини` або `Дитина 1 — ПІБ` | Перша дитина |
| `ПІБ дорослого учасника` | Дорослий учасник |
| `Дата народження учасника` | ДН дорослого або першої дитини |
| `ПІБ батька/матері або контактної особи` | Контакт для дитячої анкети |
| `Телефон для зв'язку` | Основний телефон |
| `Email батьків / дорослого учасника` | Email |
| `Telegram username або посилання на профіль` | Telegram контакт |
| `Viber контакт` | Viber контакт |
| `Резервний телефон / друга контактна особа` | Екстрений контакт |
| `Бажана група або напрям` | Куди краще додати учасника |
| `Бажана дата пробного тренування` | Якщо це новий учасник |
| `Школа / клас або рід занять` | Контекст для тренера |
| `Попередній спортивний досвід` | Досвід спорту |
| `Головна ціль тренувань` | Що очікують від занять |
| `Важливі медичні або організаційні примітки` | Що тренеру треба знати |
| `Чи можна вітати учасника з днем народження в каналі батьків?` | Дозвіл на ДН |
| `Як підписувати учасника у привітанні?` | Публічне ім'я для ДН |
| `Згода на фото/відео з тренувань` | Фото/відео дозвіл |
| `Згода на обробку персональних даних` | Згода на обробку даних |
| `Додатковий коментар` | Вільна примітка |

Поля для кількох дітей:

| Поле | Для чого |
| --- | --- |
| `Дитина 2 — ПІБ (якщо є)` | Друга дитина |
| `Дитина 2 — дата народження` | ДН другої дитини |
| `Дитина 2 — школа/клас` | Контекст для тренера |
| `Дитина 2 — медичні примітки` | Важливі обмеження |
| `Дитина 2 — ім'я для привітання з ДН` | Публічне ім'я |
| `Дитина 3 — ПІБ (якщо є)` | Третя дитина |
| `Дитина 3 — дата народження` | ДН третьої дитини |
| `Дитина 3 — школа/клас` | Контекст для тренера |
| `Дитина 3 — медичні примітки` | Важливі обмеження |
| `Дитина 3 — ім'я для привітання з ДН` | Публічне ім'я |

## Google Sheets

Основна база учасників: лист `members`.

Службовий лист `registrations` можна використовувати для ручного імпорту або контролю. У ньому має бути колонка `processed`, щоб бот міг позначати оброблені рядки у fallback-режимі.

Пріоритет fallback-листів:

1. `registration_responses`
2. `Form Responses 1`
3. `Відповіді форми 1`
4. `Ответы на форму 1`
5. `registrations`
6. `form_responses`

## Синхронізація

Власник або адміністратор може вручну запустити `/syncforms`. Автоматично бот перевіряє форми за інтервалом `FORM_POLL_INTERVAL_MINUTES`.
