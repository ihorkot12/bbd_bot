# Реєстрація членів клубу

Повна форма реєстрації потрібна для внесення учасників у базу Black Bear Dojo. Це окрема анкета для членів клубу, а не форма запису на пробне тренування.

## Жива Google Form

Поточна форма: `Black Bear Dojo 🥋 — реєстрація члена клубу`.

Посилання для батьків і дорослих учасників:
`https://docs.google.com/forms/d/e/1FAIpQLSesT2y1vreDee-V90xP66GaTEPFibCXuGI9czsOK6iqg0HpYA/viewform`

## Логіка квізу

1. Перше питання: хто реєструється.
2. Якщо це батьки, форма збирає контакти і питає, скільки дітей реєструють: `1`, `2` або `3`.
3. Після вибору кількості відкриваються тільки потрібні дитячі блоки.
4. Якщо це дорослий учасник, форма одразу веде в дорослий блок.
5. Фінальний блок спільний: поточна група або час тренувань, дозвіл на ДН-привітання, фото/відео, обробка даних і коментар.

## Дати

У формі дата народження зроблена date-полем Google Forms: користувач може вибрати дату в календарі або ввести вручну.

Клубний формат для таблиць: `ДД.ММ.РРРР`.

Бот також приймає такі формати при імпорті з таблиць:

- `РРРР-ММ-ДД`
- `ДД.ММ.РР`
- `ДД/ММ/РРРР`
- `ДД-ММ-РРРР`

У листі `members` дата народження і дата вступу записуються як `ДД.ММ.РРРР`.

## Поля для дітей

Для кожної дитини форма збирає:

- ПІБ
- дату народження
- школу/клас
- попередній спортивний досвід
- головну ціль тренувань
- медичні примітки
- ім'я для привітання з днем народження

Для гілок з кількома дітьми назви унікальні, наприклад `Дитина 1 із 2 — ПІБ` або `Дитина 3 із 3 — дата народження`. Це потрібно, щоб Google Sheets не створював дублікати заголовків.

## Поля для дорослого учасника

Форма збирає тільки базові дані для клубної бази: ПІБ, дату народження, телефон, email, Telegram, Viber, основний канал зв'язку, ціль тренувань і медичні примітки.

## Що не входить у цю форму

Повна реєстрація членів клубу не є записом на пробне тренування. У ній немає дати пробного, дати старту або продажної заявки.

Для пробного тренування використовується окрема коротка форма `TRIAL_FORM_URL`.

## Синхронізація з ботом

Основний режим читає відповіді через Google Forms API. Бот створює окремий запис у `members` для кожної дитини або одного дорослого учасника.

Повторний імпорт не дублює учасників: бот шукає вже імпортований `response_id` у нотатках і також звіряє ПІБ, дату народження та телефон.

Якщо Forms API недоступний, бот пробує fallback через листи відповідей у Google Sheets:

1. `registration_responses`
2. `Form Responses 1`
3. `Відповіді форми 1`
4. `Ответы на форму 1`
5. `registrations`
6. `form_responses`

Власник або адміністратор може вручну запустити `/syncforms`. Автоматична перевірка працює за інтервалом `FORM_POLL_INTERVAL_MINUTES`.
