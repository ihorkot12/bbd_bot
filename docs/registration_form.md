# Реєстрація членів клубу

Повна форма реєстрації потрібна для чинних або майбутніх членів Black Bear Dojo. Її можуть заповнювати батьки для 1-3 дітей або дорослий учасник для себе.

## Логіка

1. Батьки заповнюють одну форму на сім'ю.
2. Якщо дітей кілька, у формі заповнюються блоки `Дитина 1`, `Дитина 2`, `Дитина 3`.
3. Бот створює окремого учасника в `members` для кожної дитини.
4. Дорослий учасник заповнює блок дорослого, і бот створює дорослого учасника.
5. Дата народження одразу потрапляє в модуль ДН, а згода на привітання керує тим, чи бот буде готувати привітання.
6. Після обробки рядка бот ставить `processed=true`, щоб не створювати дублікати.

## Аркуш у Google Sheets

Бот спочатку читає аркуш `registrations`, потім `registration_responses`, і тільки після цього старий технічний `form_responses`.

У `registrations` мають бути такі ключові колонки:

| Колонка | Для чого |
| --- | --- |
| `registration_id` | Унікальний ID відповіді або рядка |
| `created_at` | Коли форму заповнили |
| `participant_type` | `child` або `adult` |
| `child_full_name` | ПІБ дитини, якщо один учасник в рядку |
| `adult_full_name` | ПІБ дорослого учасника |
| `birth_date` | Дата народження учасника |
| `birthday_greeting_enabled` | Чи можна вітати з ДН |
| `birthday_public_name` | Як підписати у привітанні |
| `photo_video_consent` | Згода на фото/відео |
| `parent_full_name` | ПІБ батька/матері або контактної особи |
| `parent_phone` | Телефон |
| `parent_email` | Email |
| `parent_telegram_username` | Telegram |
| `parent_viber` | Viber |
| `preferred_contact_channel` | Зручний канал зв'язку |
| `emergency_contact` | Екстрений контакт |
| `school_class_or_occupation` | Школа/клас для дитини або рід занять для дорослого |
| `previous_sport_experience` | Попередній спортивний досвід |
| `training_goal` | Мета тренувань |
| `medical_notes` | Медичні примітки |
| `preferred_group` | Бажана група/час |
| `preferred_trial_date` | Бажана дата першого тренування |
| `consent_personal_data` | Згода на обробку персональних даних |
| `notes` | Додатковий коментар |
| `processed` | Службова колонка бота |

## Поля Google Form

Рекомендована структура живої форми:

1. `Тип реєстрації`: батьки реєструють дитину/дітей або дорослий учасник.
2. `Скільки дітей реєструєте?`: 1, 2 або 3.
3. Контакт батьків/учасника: ПІБ, телефон, email, Telegram, Viber, зручний канал зв'язку.
4. Блок `Дитина 1`: ПІБ, дата народження, школа/клас, досвід спорту, мета тренувань, медичні примітки, ім'я для ДН.
5. Блок `Дитина 2`: ті самі поля, необов'язково.
6. Блок `Дитина 3`: ті самі поля, необов'язково.
7. Блок дорослого учасника: ПІБ, дата народження, рід занять, досвід спорту, мета тренувань, медичні примітки, ім'я для ДН.
8. Згоди: обробка персональних даних, фото/відео, привітання з днем народження.
9. Організаційне: бажана група/час, бажана дата першого тренування, екстрений контакт, коментар.

## Синхронізація

Власник або адміністратор може вручну запустити `/syncforms`. Автоматично бот також перевіряє форми за інтервалом `FORM_POLL_INTERVAL_MINUTES`.
